# Critical Bug Identification and Resolution

The most critical bug encountered during development was the API communication failure between frontend and backend due to Next.js proxy configuration issues. The frontend attempted to route API calls through the Next.js proxy at port 3000, but the proxy failed to forward requests to the Express backend at port 3001, resulting in 404 errors on all telemetry submissions. This bug completely blocked the core workflow, preventing users from submitting machine telemetry data and receiving health analysis results. The issue was diagnosed by testing direct API calls to the backend endpoint using curl commands, which confirmed the backend functionality was working correctly. The resolution involved modifying the frontend code to use direct backend URLs (http://localhost:3001) instead of relying on the Next.js proxy, and updating the API_BASE_URL constant in both the index.js and history.js components.

# Key Security Issue and Mitigation

The most significant security issue addressed was the prevention of SQL injection attacks through parameterized database queries. Although the application runs on localhost, implementing proper database security was essential for maintaining secure coding practices. The original database queries were vulnerable to injection through the machine_id parameter in the history retrieval endpoint. The mitigation involved converting all SQL queries to use parameterized statements with question mark placeholders, ensuring that user input is treated as data rather than executable code. This implementation prevents malicious SQL injection attempts while maintaining the application's functionality. The security measure was particularly important because it establishes a secure foundation that would be critical if the application were ever deployed beyond the localhost environment.
